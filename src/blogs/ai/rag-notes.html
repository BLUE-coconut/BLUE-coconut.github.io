RAG（Retrieval-Augmented Generation）是工程上最实用的大模型应用模式之一。 本文记录系统设计中的关键决策点。

文档切分策略

切分粒度直接影响召回质量。常见策略：

  • 固定窗口：每 500 token 一段，重叠 50 token。简单但可能切碎语义。
  • 按段落切分：以 \n\n 为分隔。保留语义完整性。
  • 结构化切分：按 Markdown 标题或 HTML 标签。适合有清晰层级的文档。

Embedding 模型选择

中文场景下推荐 BAAI/bge-large-zh-v1.5 或 text-embedding-3-small（OpenAI）。bge 开源免费，但需自托管。

Re-rank 提升精度

向量检索召回 top-50 后，再用 cross-encoder 重新排序（top-5）。 典型提升：NDCG@10 从 0.65 提升到 0.82。

评估方法

用 LLM-as-judge 评估回答质量，准备 50-100 个真实 query 集。 关键指标：忠实度（是否编造）、召回率、答案相关性。
