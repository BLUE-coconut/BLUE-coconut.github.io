强化学习（Reinforcement Learning，RL）是机器学习的一个重要分支：序贯决策任务的机器学习方法——强化学习（涉及多轮交互）。 本文整理其基础概念、关键要素、与监督学习的区别，以及常见的学习范式分类。

一、强化学习的定义

强化学习是机器通过与环境交互来实现目标的一种计算方法。

因为决策任务是多轮的，智能体就需要在每轮做决策时考虑未来环境相应的改变，而不是单步最优决策。

智能体（Agent）

强化学习用「智能体（agent）」这个概念来表示做决策的机器。

  • 每一轮交互：智能体在环境中的某一状态下做出动作决策，把这个动作用到环境中，环境发生相应的改变，并将相应的反馈和下一轮状态传回机器。
  • 优化目标：最大化在多轮交互中获得累计奖励反馈的期望

相比于有监督学习中的「模型」，强化学习中的「智能体」强调机器不但可以感知周围的环境信息，还可以通过做决策来直接改变这个环境， 而不只是给出一些预测信号。

智能体的关键要素

智能体的关键要素可以概括为三个：感知、行动、目标。

二、强化学习系统的要素

历史（History）

历史是观察、行动和奖励的序列，即智能体到目前为止所经历的全部信息。 是强化学习最原始的视角——所有后续概念都是对历史的压缩。

状态（State）

状态是关于历史的函数，用于确定接下来会发生的事情（行动、观察、奖励）的信息。 状态是对历史的提炼，是决策与预测的基础。

策略（Policy）

策略是从状态到行动的映射 A = F(S)，用来学习智能体在特定时间的行为方式。 策略有两种基本形式：

  • 确定性策略：给定状态输出唯一动作
  • 随机策略：给定状态输出动作的概率分布

奖励（Reward）

一个定义强化学习目标的标量 R(s, a)。 奖励是即时的反馈信号，用来评价当前这一步动作的好坏。

价值函数（Value Function）

价值函数也是一个标量，用于对未来累积奖励的预测（对于长期来说什么是好的）。 与奖励的区别在于：奖励是即时的，价值是对未来收益的估计。

三、强化学习与有监督学习的区别

维度有监督学习强化学习
训练数据分布固定不变随智能体的学习而改变
数据来源预先标注的样本智能体与环境交互产生
优化目标在给定数据分布下使损失函数期望最小在交互数据分布下使奖励期望最大
反馈形式标签（正确答案）奖励（仅评价，不直接告诉怎么做）

占用度量（关于 RL 中数据分布的一个概念）

归一化的占用度量用于衡量在一个智能体决策与一个动态环境的交互过程中， 采样到一个具体的状态动作对（state-action pair）的概率分布。

  • 性质：给定两个策略及其与一个动态环境交互得到的两个占用度量，那么当且仅当这两个占用度量相同时，这两个策略相同。
  • 由性质推导出强化学习的本质：
    • 强化学习的一大难点就在于，智能体看到的数据分布是随着智能体的学习而不断发生改变的。
    • 占有度量是状态动作对的概率分布，而策略是动作到状态的映射。对智能体的策略进行优化的目标是使其奖励最大化，可以表达为一个占有度量下对应的奖励的期望。

范式对比总结

  • 一般的有监督学习关注寻找一个模型，使其在给定数据分布下得到的损失函数的期望最小。（数据分布确定）
  • 强化学习关注寻找一个智能体策略，最大化其与环境交互的过程所产生的数据分布下，给定奖励函数的期望。（目标函数确定）

四、强化学习分类

按照环境是否已知划分：免模型 vs 有模型

类别核心思想典型算法
免模型学习（Model-Free） 不去学习和理解环境，环境给出什么信息就是什么信息 Policy Optimization、Q-learning
有模型学习（Model-Based） 去学习和理解环境，学会用一个模型来模拟环境，通过模拟的环境来得到反馈 World Models、AlphaZero 等

一般情况下，环境都是不可知的，所以实践中主要研究无模型问题。

按照学习方式划分：在线策略 vs 离线策略

类别特征典型算法
在线策略（On-Policy） agent 必须本人在场，并且一定是本人边玩边学习 Sarsa
离线策略（Off-Policy） agent 可以选择自己玩，也可以选择看着别人玩，从过往的经验中学习。过往的经历没必要是自己的，玩和学习的时间可以不同步 Q-learning、Deep Q Network

按照学习目标划分：基于策略 vs 基于价值

类别输出适用动作空间典型算法
基于策略（Policy-Based） 下一步动作的概率，根据概率来选取动作 非连续和连续 Policy Gradients
基于价值（Value-Based） 动作的价值，选择价值最高的动作 非连续 Q-learning、Deep Q Network、Sarsa
Actor-Critic（二者结合） Actor 根据概率做出动作，Critic 根据动作给出价值，加速学习 连续 / 非连续 A2C、A3C、DDPG

小结

强化学习与有监督学习的根本分歧在于：数据分布是固定的还是随智能体变化的。 这一分歧导致强化学习的所有概念（策略、价值、占用度量）都围绕「在变化的数据分布下优化长期奖励」这一目标展开。

实际工程中，往往需要把多种范式组合使用：例如 Actor-Critic 架构结合了 Policy-Based 的动作采样能力与 Value-Based 的长期评估能力， DQN 则是 Value-Based + 神经网络 + 经验回放的经典组合。理解这些范式的核心动机比记住具体公式更重要。
