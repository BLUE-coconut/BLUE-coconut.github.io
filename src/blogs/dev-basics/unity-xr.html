二、XR Interaction Toolkit（XRI）

XR Origin（玩家原点）

XR Origin 是 XR 场景中代表玩家的坐标原点，承载相机、手柄等设备的位置与朝向。 所有的交互（射线、抓取、传送）都基于 XR Origin 坐标系计算。

输入系统

Unity 推荐使用新的 Input System Package 来处理 XR 设备输入：

  • Unity Input System 使用方法：官方文档
  • Unity XR 输入设备信息：官方文档
  • Unity VR + XRI 3 输入系统配置：CSDN 博客

在 XR 场景中，常见的输入需求包括：

  • 手柄按钮（Trigger / Grip / Menu）
  • 手柄摇杆（Thumbstick）
  • 手柄位姿（Position + Rotation）
  • 手势识别（Hand Tracking）

HTC VIVE Basic Input for OpenXR

通过 OpenXR 接入 VIVE 头显与手柄，参考：HTC VIVE Basic Input for OpenXR。

XRI（XR Interaction Toolkit）封装了常见的 XR 交互模式：近距抓取（Near Grab）、远距射线抓取（Far Ray Grab）、UI 交互、传送（Teleport）等， 大幅简化了 XR 应用开发。

三、Input System（输入系统）

Input Action Asset

建议使用 Input Action Asset（.inputactions 文件）来统一管理输入配置，把输入「数据化」：

  • Action Map：按场景/角色分组（如 Player、UI）
  • Action：具体的输入动作（如 Jump、Move）
  • Binding：动作到具体输入设备的映射（如键盘 Space、XRI 手柄 Trigger）

XR 输入类型

输入类型XR 中的获取方式典型用途
Vector2Thumbstick / Touchpad移动、转向
ButtonTrigger / Grip / Face Button抓取、确认、射击
Pose设备位姿 (Position + Rotation)射线、手部 IK
Trigger (float)扳机按压程度力度交互、握力反馈

四、Unity 物理模拟

产生碰撞的条件

  1. 若要产生碰撞，双方都要有碰撞器
  2. 运动的一方一定要有刚体，另一方有无刚体无所谓

注：如果运动的一方无刚体，它去碰撞静止的刚体，相当于没撞上。

接触的两种方式

  1. Collision 碰撞：造成物理碰撞，可以在碰撞时执行 OnCollision 事件
  2. Trigger 触发：取消所有的物理碰撞，可以在触发时执行 OnTrigger 事件

注：两个物体接触不可能同时产生碰撞 + 接触，最多产生一种。但是可以 A、B 产生碰撞，A、C 产生触发。

产生不同方式接触的条件

方式条件
Collision 碰撞 1. 双方都有碰撞体
2. 运动的一方必须有刚体
3. 双方不可同时勾选 Kinematic 运动学
4. 双方都不可勾选 Trigger 触发器
Trigger 触发 1. 双方都有碰撞体
2. 运动的一方必须是刚体
3. 至少一方勾选 Trigger 触发器

五、XR 开发小贴士

  • 性能优先：XR 应用对帧率要求严格（一般要求 90 FPS），场景多边形数、shader 复杂度都要严格控制
  • 空间音频：XR 体验强烈依赖空间音频，建议尽早集成 AudioSource 的 Spatial Blend 设置
  • 舒适度：避免强制瞬移、避免持续高速运动、避免立体视觉冲突
  • 输入抽象：用 Input Action Asset 抽象所有输入，方便跨平台（Quest / Vision Pro / PC VR）切换
