Flutter 用一套 Dart 代码同时构建 iOS、Android、Web、桌面应用。本文总结关键设计模式。

Widget 体系

Flutter 一切皆 Widget，分为三类：

  • StatelessWidget：不可变，纯渲染
  • StatefulWidget：可变状态，配合 State 类使用
  • InheritedWidget：依赖注入，状态向下传递

状态管理

主流方案对比：

  • Provider：官方推荐，入门首选
  • Riverpod：Provider 的进化版，编译期安全
  • Bloc：事件驱动，适合复杂业务
  • GetX：轻量但社区有争议

布局核心

理解三个核心 Widget：

Column(children: [...])  // 纵向排列
Row(children: [...])     // 横向排列
Stack(children: [...])   // 层叠

性能优化

  • 用 const 标记不变 Widget，让框架跳过重建
  • 长列表用 ListView.builder（懒构建）
  • 避免在 build 中执行耗时操作
